TIP OF THE DAY

Zero-width Unicode characters (U+200B ZWSP, U+200C ZWNJ, U+200D ZWJ, U+FEFF BOM,
U+2060 WJ) are completely invisible in every common code editor, diff view, and
markdown renderer — but language models read and execute them as instructions.
TrapDoor proves this is a real attack, not a theoretical one. For any AI-assisted
project, `cat -v` and `hexdump -C` are your friends for inspecting AI config files.
Add this one-liner to your git pre-commit hooks now:
  grep -qrP '[\x{200b}\x{200c}\x{200d}\x{feff}\x{2060}]' \
    --include="CLAUDE.md" --include=".cursorrules" . && \
    echo "BLOCKED: zero-width Unicode chars detected in AI config files" && exit 1
