- Category: AI Dev > IDE
- Impact: 🔴 Directly affects us (we use Cursor AI)
- Summary: An attacker embeds a bare repository containing a malicious
  pre-commit Git hook inside an otherwise legitimate-looking repo. When
  Cursor AI's agent runs `git checkout` as part of routine work, the hook

  cloned repo before running agent commands. Inspect `.git/hooks/` and all
  nested directories on any unknown repo.
- Source: [Novee Security](https://novee.security/blog/cursor-ide-cve-2026-26268-git-hook-arbitrary-code-execution/),

TIP OF THE DAY
Git hooks are the new `.cursorrules` — an underestimated local-execution
surface hiding in repositories. Three active threats this week exploit them:
CVE-2026-26268 (Cursor), Lazarus Contagious Interview, and any copycat using
the now-public Shai-Hulud kit.

Set this as a global git default today:
  git config --global core.hooksPath /dev/null

Then consciously re-enable hooks only in repos you own and trust:
  git config core.hooksPath .git/hooks  # run inside the trusted repo

This one command eliminates the entire git hook attack surface for cloned
repos without affecting your daily workflow in your own projects.
