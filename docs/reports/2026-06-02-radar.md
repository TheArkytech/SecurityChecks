TIP OF THE DAY

The codexui-android attack hid its credential-stealing code ONLY in the published npm tarball,
not in the GitHub repo. This defeats the common "check the source code" audit step.
To catch this class of attack, verify that published npm packages match their source tags:
  npm pack <package-name>@<version> --dry-run
  Compare the tarball file list against what appears in the GitHub release tag.
Any file present in the tarball but absent from the GitHub repo is a red flag.
Tools like Socket.dev and Aikido.dev automate this diff at install time.
